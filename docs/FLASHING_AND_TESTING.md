# Flashing & Testing Guide

Руководство по сборке, прошивке и тестированию прошивки Hoverboard-Firmware-Hack-Gen2-FOC.

---

## Что понадобится

**Железо:**
- ST-Link/V2 (или отладочная плата STM32 со встроенным ST-Link)
- Кабель питания ховерборда (36V аккумулятор или лабораторный БП 30–42V, ≥3A)
- Провода для SWD: GND, SWDIO, SWCLK (3 провода, 3.3V — НЕ подключать питание от ST-Link)

**Программы:**
- Keil MDK 5 (бесплатно до 32KB, [keil.com/mdk5](https://www.keil.com/download/product/)) — для сборки и прошивки
- STM32 ST-LINK Utility (опционально, для разблокировки flash)

**Проект:** `HoverBoardGigaDevice/Hoverboard.uvprojx`

---

## Разъём для прошивки

На каждой плате есть 4-пиновый разъём SWD (рядом с процессором):

```
[ GND | SWCLK | SWDIO | 3V3 ]
```

Подключить ST-Link:
- GND → GND
- SWDIO → SWDIO
- SWCLK → SWCLK
- **3V3 НЕ подключать** — плата питается от аккумулятора

---

## Первая прошивка (разблокировка Flash)

GD32F130C8 с завода поставляется с заблокированным Flash. Разблокировать один раз:

1. Открыть **STM32 ST-LINK Utility**
2. Подключиться: `Target → Connect`
3. Если появляется ошибка `"Access denied"` → `Target → Option Bytes`
4. В поле `Read Out Protection` выбрать `Disabled` → Apply
5. Flash разблокирован, теперь можно прошивать через Keil

## Сборка в Keil

### Выбор роли платы

В файле `HoverBoardGigaDevice/Inc/config.h` строка 38–39:

```c
#define MASTER    // для платы с кнопкой питания, дисплеем, управлением
//#define SLAVE   // для второй платы (только получает PWM команды)
```

Определить какая плата какая: MASTER — та, к которой подключена кнопка питания.

### Сборка

1. Открыть `HoverBoardGigaDevice/Hoverboard.uvprojx`
2. Установить нужный `#define` (MASTER или SLAVE) в `config.h`
3. `Project → Build Target` (F7)
4. В Output Window должно быть `0 Error(s), 0 Warning(s)`

### Включение синусоидальной коммутации

В `config.h` раскомментировать (после строки SINUSOIDAL):

```c
#define SINUSOIDAL
```

Закомментировать — вернётся блочная коммутация. Пересобрать.

## Прошивка

> **ВАЖНО:** Держать кнопку питания ховерборда нажатой на всё время прошивки. MCU управляет силовым реле — при прошивке он отпускает его и плата обесточивается.

1. Подключить ST-Link к SWD разъёму платы
2. Включить питание ховерборда (нажать и удерживать кнопку)
3. В Keil: `Flash → Download` (F8)
4. Дождаться `"Flash Load finished at ..."`
5. Отпустить кнопку

Повторить для второй платы (переключив MASTER/SLAVE в config.h).

---

## Проверки после прошивки

### 1. Базовая — включение

1. Нажать кнопку питания (короткое нажатие)
2. Услышать **звуковой сигнал** (нисходящая гамма при старте) — MASTER жив
3. Светодиод должен загореться зелёным (батарея >35V) или оранжевым

Если тишина и ничего не светится — скорее всего прошита не та плата или MASTER/SLAVE перепутаны.

### 2. Связь MASTER ↔ SLAVE

Обе платы должны быть прошиты и подключены друг к другу через USART1 (PA2/PA3).

Проверка: подать небольшую команду скорости (~200/1000) через управляющее устройство. Оба мотора должны реагировать. Если крутится только один — проблема с USART между платами.

### 3. Тест синусоидальной коммутации

**Что ожидать:**
- Мотор вращается **плавно**, без ступенчатых рывков
- Работает **тише** чем при блочной коммутации
- Ток при холостом ходу примерно такой же или меньше

**Как проверить вращение:**
1. Поднять ховерборд на подставку (колёса в воздухе)
2. Дать сигнал скорости ~200–300 (30% от максимума)
3. Мотор должен набрать обороты плавно

**Сравнение с блочной коммутацией:**
Закомментировать `#define SINUSOIDAL` → пересобрать → перепрошить. Разница в шуме и плавности хорошо слышна.

## Калибровка направления вращения

Если мотор крутится в **обратную сторону** при положительной скорости:

В `HoverBoardGigaDevice/Src/bldc.c` найти таблицу `pos_to_angle` и заменить на инвертированный вариант:

```c
// Текущий (прямое направление):
static const uint16_t pos_to_angle[8] = {
    0,      // [0] invalid
    0,      // [1] pos1 → 0°
    10923,  // [2] pos2 → 60°
    21845,  // [3] pos3 → 120°
    32768,  // [4] pos4 → 180°
    43691,  // [5] pos5 → 240°
    54613,  // [6] pos6 → 300°
    0,      // [7] invalid
};

// Обратное направление (если мотор едет не туда):
static const uint16_t pos_to_angle[8] = {
    0,      // [0] invalid
    0,      // [1] pos1 → 0°
    54613,  // [2] pos2 → 300°
    43691,  // [3] pos3 → 240°
    32768,  // [4] pos4 → 180°
    21845,  // [5] pos5 → 120°
    10923,  // [6] pos6 → 60°
    0,      // [7] invalid
};
```

Пересобрать и перепрошить.

## Параметры настройки (config.h)

| Параметр | Значение по умолчанию | Описание |
|---|---|---|
| `SINUSOIDAL` | определён | Синусоидальная коммутация. Закомментировать для блочной |
| `DC_CUR_LIMIT` | 15A | Ограничение тока. Уменьшить до 10A для первых тестов |
| `FILTER_SHIFT` | 12 | Сглаживание ШИМ. Меньше = быстрее реакция, больше = плавнее |
| `TIMEOUT_MS` | 2000 | Таймаут без команд (мс) перед отключением |
| `BAT_LOW_DEAD` | 31.0V | Нижний порог батареи для отключения |

## Типичные проблемы

| Симптом | Причина | Решение |
|---|---|---|
| Плата не прошивается | Flash заблокирован | Разблокировать через ST-LINK Utility |
| Нет звука при включении | Прошита SLAVE вместо MASTER | Проверить `#define MASTER` в config.h |
| Мотор дёргается, не крутится | Неправильная таблица Холла | Инвертировать `pos_to_angle` |
| Мотор греется в покое | Слишком высокий `DC_CUR_LIMIT` | Уменьшить до 5–10A |
| Один мотор не работает | Нет связи MASTER↔SLAVE | Проверить подключение PA2/PA3 между платами |
| Плата отключается при прошивке | Кнопка питания не удерживается | Держать кнопку всё время прошивки |
