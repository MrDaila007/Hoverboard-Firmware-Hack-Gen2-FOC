# CI Build Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Настроить GitHub Actions CI для автоматической сборки прошивки GD32F130C8 в конфигурациях MASTER и SLAVE с выгрузкой `.elf`/`.bin` артефактов.

**Architecture:** Makefile с `arm-none-eabi-gcc` обеспечивает сборку локально и в CI. GitHub Actions скачивает CMSIS-5 и GD32F1x0 Firmware Library по фиксированным тегам, запускает `make ROLE=MASTER` и `make ROLE=SLAVE` параллельно, выгружает артефакты. Linker script задаёт карту памяти GD32F130C8 (FLASH 64K @ 0x08000000, RAM 8K @ 0x20000000).

**Tech Stack:** arm-none-eabi-gcc, GNU Make, GitHub Actions, CMSIS-5 v5.9.0, GD32F1x0 Firmware Library v3.1.0.

## Task 1: Linker script

**Files:**
- Create: `HoverBoardGigaDevice/Linker/gd32f130c8.ld`

- [ ] **Шаг 1.1: Создать директорию и файл**

```bash
mkdir -p HoverBoardGigaDevice/Linker
```

Создать `HoverBoardGigaDevice/Linker/gd32f130c8.ld`:

```ld
/* GD32F130C8: 64KB Flash, 8KB RAM */
MEMORY
{
  FLASH (rx)  : ORIGIN = 0x08000000, LENGTH = 64K
  RAM   (rwx) : ORIGIN = 0x20000000, LENGTH = 8K
}

ENTRY(Reset_Handler)

SECTIONS
{
  /* Interrupt vector table must be first */
  .isr_vector :
  {
    KEEP(*(.isr_vector))
  } > FLASH

  /* Code and read-only data */
  .text :
  {
    *(.text*)
    *(.rodata*)
    _etext = .;
  } > FLASH

  /* Initialized data: stored in FLASH, copied to RAM at startup */
  _sidata = LOADADDR(.data);
  .data :
  {
    _sdata = .;
    *(.data*)
    _edata = .;
  } > RAM AT > FLASH

  /* Zero-initialized data */
  .bss :
  {
    _sbss = .;
    *(.bss*)
    *(COMMON)
    _ebss = .;
  } > RAM

  /DISCARD/ : { *(.eh_frame) }
}
```

## Task 3: Makefile

**Files:**
- Create: `Makefile`

Использовать пути из Task 2. Если фактическая структура GD32 библиотеки отличается от ожидаемой — скорректировать переменные `GD32_INC_DEVICE` и `GD32_INC_PERIPH`.

- [ ] **Шаг 3.1: Создать Makefile**

```makefile
# Role: MASTER or SLAVE (default: MASTER)
ROLE ?= MASTER
BUILD_DIR = build/$(ROLE)
TARGET    = $(BUILD_DIR)/hoverboard-$(ROLE)

CC      = arm-none-eabi-gcc
OBJCOPY = arm-none-eabi-objcopy
SIZE    = arm-none-eabi-size

# External library paths — set via environment or override on command line
CMSIS_PATH ?= $(error Set CMSIS_PATH to the CMSIS_5 root directory)
GD32_PATH  ?= $(error Set GD32_PATH to the GD32F1x0_Firmware_Library root directory)

# Compiler flags
CFLAGS  = -mcpu=cortex-m3 -mthumb -mfloat-abi=soft
CFLAGS += -Os -std=c99 -Wall -Wextra -Wno-unused-parameter
CFLAGS += -ffunction-sections -fdata-sections
CFLAGS += -D$(ROLE) -DARM_MATH_CM3 -DUSE_STDPERIPH_DRIVER

# Include paths
INCLUDES  = -IHoverBoardGigaDevice/Inc
INCLUDES += -IHoverBoardGigaDevice/RTE/Device/GD32F130C8
INCLUDES += -IHoverBoardGigaDevice/RTE/_Target_1
INCLUDES += -I$(CMSIS_PATH)/CMSIS/Core/Include
INCLUDES += -I$(CMSIS_PATH)/CMSIS/DSP/Include
INCLUDES += -I$(GD32_PATH)/Firmware/CMSIS/GD/GD32F1x0/Include
INCLUDES += -I$(GD32_PATH)/Firmware/GD32F1x0_standard_peripheral/Include

# Source files
C_SRCS  = $(wildcard HoverBoardGigaDevice/Src/*.c)
C_SRCS += $(wildcard HoverBoardGigaDevice/RTE/Device/GD32F130C8/*.c)
ASM_SRCS = HoverBoardGigaDevice/RTE/Device/GD32F130C8/startup_gd32f1x0.s

C_OBJS   = $(patsubst %.c,  $(BUILD_DIR)/%.o, $(C_SRCS))
ASM_OBJS = $(patsubst %.s,  $(BUILD_DIR)/%.o, $(ASM_SRCS))
OBJS     = $(C_OBJS) $(ASM_OBJS)

# Linker flags
LDFLAGS  = -mcpu=cortex-m3 -mthumb -mfloat-abi=soft
LDFLAGS += -THoverBoardGigaDevice/Linker/gd32f130c8.ld
LDFLAGS += -Wl,--gc-sections
LDFLAGS += -Wl,-Map=$(TARGET).map
LDFLAGS += -specs=nosys.specs

.PHONY: all clean

all: $(TARGET).elf $(TARGET).bin
	@$(SIZE) $<

$(TARGET).elf: $(OBJS)
	$(CC) $(LDFLAGS) $^ -o $@

$(TARGET).bin: $(TARGET).elf
	$(OBJCOPY) -O binary $< $@

$(BUILD_DIR)/%.o: %.c
	@mkdir -p $(dir $@)
	$(CC) $(CFLAGS) $(INCLUDES) -c $< -o $@

$(BUILD_DIR)/%.o: %.s
	@mkdir -p $(dir $@)
	$(CC) $(CFLAGS) $(INCLUDES) -c $< -o $@

clean:
	rm -rf build/
```

## Task 4: GitHub Actions workflow

**Files:**
- Create: `.github/workflows/build.yml`

- [ ] **Шаг 4.1: Создать директорию и файл**

```bash
mkdir -p .github/workflows
```

Создать `.github/workflows/build.yml`:

```yaml
name: Build Firmware

on:
  push:
    branches:
      - master
      - 'feature/**'
  pull_request:
    branches:
      - master

jobs:
  build:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        role: [MASTER, SLAVE]
      fail-fast: false

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install ARM GCC toolchain
        run: |
          sudo apt-get update -q
          sudo apt-get install -y --no-install-recommends \
            gcc-arm-none-eabi \
            binutils-arm-none-eabi

      - name: Download CMSIS-5
        run: |
          curl -sL \
            https://github.com/ARM-software/CMSIS_5/archive/refs/tags/5.9.0.tar.gz \
            | tar -xz -C $HOME
          echo "CMSIS_PATH=$HOME/CMSIS_5-5.9.0" >> $GITHUB_ENV

      - name: Download GD32F1x0 Firmware Library
        run: |
          curl -sL \
            https://github.com/GigaDevice-Semiconductor/GD32F1x0_Firmware_Library/archive/refs/tags/V3.1.0.tar.gz \
            | tar -xz -C $HOME
          echo "GD32_PATH=$HOME/GD32F1x0_Firmware_Library-3.1.0" >> $GITHUB_ENV

      - name: Build ${{ matrix.role }}
        run: make ROLE=${{ matrix.role }}

      - name: Upload firmware artifacts
        uses: actions/upload-artifact@v4
        with:
          name: firmware-${{ matrix.role }}
          path: |
            build/${{ matrix.role }}/hoverboard-${{ matrix.role }}.elf
            build/${{ matrix.role }}/hoverboard-${{ matrix.role }}.bin
          retention-days: 30
```

- [ ] **Шаг 5.3: Исправить ошибки компиляции если CI упал**

Типичные причины:

**`gd32f1x0.h: No such file`** — фактический путь GD32 библиотеки отличается. Запустить:
```bash
# Локально проверить структуру
find $HOME/GD32F1x0_Firmware_Library-3.1.0 -name "gd32f1x0.h" 2>/dev/null
```
Исправить `GD32_INC_DEVICE` в Makefile.

**`gd32f1x0_libopt.h: No such file`** — создать файл по шагу 2.3 выше (если не был создан).

**`arm_math.h: No such file`** — путь CMSIS DSP неверный. Проверить:
```bash
find $HOME/CMSIS_5-5.9.0 -name "arm_math.h"
```
Исправить `-I$(CMSIS_PATH)/CMSIS/DSP/Include` в Makefile.

**`undefined reference to Reset_Handler`** — startup файл не скомпилировался. Проверить что `startup_gd32f1x0.s` компилируется как `.s` (не `.S`). Если расширение `.S` — исправить `ASM_SRCS` в Makefile.
