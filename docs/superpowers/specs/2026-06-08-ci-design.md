# CI Build Pipeline — Design Spec

**Date:** 2026-06-08
**Project:** Hoverboard-Firmware-Hack-Gen2-FOC
**Target:** GD32F130C8 (ARM Cortex-M3, 64KB Flash, 8KB RAM)

---

## Goal

Автоматическая сборка прошивки в двух конфигурациях (MASTER и SLAVE) на каждый PR и пуш. Артефакты `.elf` и `.bin` загружаются в GitHub и доступны 30 дней.

---

## Новые файлы

| Файл | Назначение |
|---|---|
| `Makefile` | Сборка MASTER/SLAVE через arm-none-eabi-gcc |
| `.github/workflows/build.yml` | GitHub Actions workflow |
| `HoverBoardGigaDevice/Linker/gd32f130c8.ld` | Linker script: FLASH 64K @ 0x08000000, RAM 8K @ 0x20000000 |

---

## Makefile

**Параметр:** `ROLE=MASTER` или `ROLE=SLAVE` (по умолчанию MASTER).

**Выходные файлы:** `build/$(ROLE)/hoverboard-$(ROLE).elf` и `.bin`.

**Флаги компилятора:**
- `-mcpu=cortex-m3 -mthumb -mfloat-abi=soft`
- `-Os -std=c99 -Wall -Wextra`
- `-ffunction-sections -fdata-sections` (удаление неиспользуемого кода)
- `-D$(ROLE) -DARM_MATH_CM3 -DUSE_STDPERIPH_DRIVER`

**Исходники:**
- `HoverBoardGigaDevice/Src/*.c`
- `HoverBoardGigaDevice/RTE/Device/GD32F130C8/*.c`
- `HoverBoardGigaDevice/RTE/Device/GD32F130C8/startup_gd32f1x0.s`

**Include paths** (два внешних пути задаются переменными окружения `CMSIS_PATH` и `GD32_PATH`):
- `HoverBoardGigaDevice/Inc`
- `HoverBoardGigaDevice/RTE/Device/GD32F130C8`
- `HoverBoardGigaDevice/RTE/_Target_1`
- `$(CMSIS_PATH)/CMSIS/Core/Include`
- `$(CMSIS_PATH)/CMSIS/DSP/Include`
- `$(GD32_PATH)/Firmware/CMSIS/GD/GD32F1x0/Include`
- `$(GD32_PATH)/Firmware/GD32F1x0_standard_peripheral/Include`

**Флаги линковщика:**
- `-THoverBoardGigaDevice/Linker/gd32f130c8.ld`
- `-Wl,--gc-sections` (убрать неиспользуемые секции)
- `-specs=nosys.specs` (без OS-зависимых syscalls)

**Локальный запуск:**
```bash
CMSIS_PATH=~/cmsis GD32_PATH=~/gd32sdk make ROLE=MASTER
make ROLE=SLAVE
make clean
```

## GitHub Actions Workflow (`.github/workflows/build.yml`)

**Триггеры:**
- `push` на ветки `master` и `feature/**`
- `pull_request` в `master`

**Матрица:** `role: [MASTER, SLAVE]` — два джоба выполняются параллельно.

**Шаги:**
1. `actions/checkout@v4`
2. `apt-get install gcc-arm-none-eabi binutils-arm-none-eabi`
3. Скачать CMSIS-5 тег `5.9.0` с `github.com/ARM-software/CMSIS_5`, записать `CMSIS_PATH` в `$GITHUB_ENV`
4. Скачать GD32F1x0 Firmware Library тег `V3.1.0` с `github.com/GigaDevice-Semiconductor/GD32F1x0_Firmware_Library`, записать `GD32_PATH` в `$GITHUB_ENV`
5. `make ROLE=${{ matrix.role }}`
6. `actions/upload-artifact@v4` — загрузить `.elf` и `.bin`, `retention-days: 30`

**Артефакты:** `firmware-MASTER` и `firmware-SLAVE`.

## Linker Script (`gd32f130c8.ld`)

Карта памяти:
- `FLASH (rx)`: `0x08000000`, 64K — код и read-only данные
- `RAM (rwx)`: `0x20000000`, 8K — стек, .data, .bss

Секции: `.text` (код + rodata) → FLASH; `.data` (инициализированные переменные) → RAM, загружается из FLASH; `.bss` (нулевые переменные) → RAM.

Entry point: `Reset_Handler` (определён в `startup_gd32f1x0.s`).

---

## Ограничения

- Тестов нет — CI проверяет только успешность компиляции и линковки.
- Заголовки CMSIS и GD32 скачиваются при каждом запуске (~10 МБ, ~15 сек). Можно кешировать через `actions/cache` в будущем.
- `.bin` не верифицируется на железе в CI — только артефакт для ручной прошивки.
